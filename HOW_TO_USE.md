# How to Use This Roadmap

## Overview

This repository is your personal learning management system for becoming a protocol engineer. Everything is designed to be easy to update as you learn and build.

---

## Getting Started

### 1. Set Your Goals
**File:** `GOALS.md`

Start by defining what you want to achieve:
- Update long-term vision
- Set 6-month goals
- Define monthly objectives

**Do this now!** Fill in your current goals.

### 2. Update Your Progress Tracker
**File:** `README.md`

The main README is your dashboard:
- Update "Current Focus" section weekly
- Track your skills matrix monthly
- List active projects
- Update recent milestones

**Keep this current** - it's your snapshot of progress.

### 3. Customize Your Learning Path
**File:** `roadmap/00-learning-path.md`

This is your curriculum:
- Check off completed topics
- Update status for each phase
- Add notes on what you're learning
- Adjust the path based on your needs

## Daily/Weekly Workflow

### When You Learn Something New

1. **Find the relevant roadmap file**
   - Located in `roadmap/[category]/[topic].md`
   - Example: Learning about Merkle trees? â†’ `roadmap/01-foundations/data-structures.md`

2. **Add your notes**
   ```
   - Fill in the concepts section
   - Add code examples
   - Document insights
   - Add questions
   ```

3. **Update exercises**
   - Mark exercises as complete
   - Add learnings from each exercise

### When You Read a Paper

1. **Go to** `resources/papers/must-read.md`
2. **Find or add the paper**
3. **Use the template** to document:
   - Key contributions
   - Your notes
   - Questions
   - How it relates to your learning

### When You Study Code

1. **Go to** `resources/repositories.md`
2. **Add the repository** using the template
3. **Document:**
   - What you learned
   - Interesting patterns
   - Architecture insights
   - Questions

### When You Build Something

1. **Create project folder** in `projects/[project-name]/`
2. **Use README template** to document:
   - What you built
   - Challenges faced
   - Learnings
   - Future improvements

3. **Update main README** with project status

### When You Discover a Pattern

1. **Go to** `code-patterns/[language]/`
2. **Add the pattern** with:
   - Use case
   - Code example
   - When to use it
   - Trade-offs

---

## Weekly Reflection

**Every Sunday (or your chosen day):**

1. **Create new weekly file**
   - Copy: `progress/weekly/TEMPLATE-weekly.md`
   - Rename: `progress/weekly/2025-week-XX.md`

2. **Fill it out:**
   - What you learned
   - What you built
   - Challenges faced
   - Resources consumed
   - Next week's goals

**Why this matters:** Weekly reviews help you see patterns, celebrate wins, and adjust your approach.

---

## Monthly Review

**End of each month:**

1. **Create monthly review**
   - Copy: `progress/monthly-reviews/TEMPLATE-monthly.md`
   - Rename: `progress/monthly-reviews/2024-11.md`

2. **Complete the review:**
   - Goal achievement
   - Skills progress
   - Projects summary
   - Major learnings
   - Next month's goals

3. **Update main README** with:
   - New skills ratings
   - Completed milestones
   - Updated focus

**Why this matters:** Monthly reviews show your growth trajectory and keep you aligned with long-term goals.

## File Organization

### Roadmap Files (`roadmap/`)
```
Purpose: Detailed notes on each topic you're learning
When to update: As you learn each concept
How to use: Fill in sections as you understand them
```

**Structure:**
- Concepts explanation
- Code examples
- Practical exercises
- Resources
- Questions

### Project Files (`projects/`)
```
Purpose: Document your builds
When to update: During and after building
How to use: Track learnings and challenges
```

**Each project should have:**
- README with overview
- Challenges and solutions
- Key learnings
- Future improvements

### Progress Files (`progress/`)
```
Purpose: Track your journey over time
When to update: Weekly and monthly
How to use: Reflect on progress and plan ahead
```

**Contains:**
- Weekly reflections
- Monthly reviews
- Skills tracking

### Resources Files (`resources/`)
```
Purpose: Track external learning resources
When to update: As you discover and consume them
How to use: Document papers, repos, talks, etc.
```

**Includes:**
- Papers to read
- Repositories to study
- Talks and videos
- Books and courses

### Code Patterns (`code-patterns/`)
```
Purpose: Your personal pattern library
When to update: When you discover reusable patterns
How to use: Reference when coding similar problems
```

### Experiments (`experiments/`)
```
Purpose: Small learning projects
When to update: When doing experiments
How to use: Quick tests and explorations
```

## Tips for Success

### 1. Be Consistent
- Update weekly, even if brief
- Small, regular updates > rare large updates
- Make it a habit

### 2. Be Honest
- Document challenges, not just wins
- Note what doesn't work
- Track areas needing improvement

### 3. Be Practical
- Focus on understanding, not just checking boxes
- Add real code examples from your work
- Connect concepts across topics

### 4. Be Reflective
- Review your notes regularly
- See patterns in your learning
- Adjust your approach based on what works

### 5. Be Forward-Looking
- Set clear next steps
- Plan your learning path
- Update goals as you grow
